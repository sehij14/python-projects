## Day 15 - Company Management System

Today’s project was focused on understanding Polymorphism and Method Overriding in Object-Oriented Programming.

Beyond creating basic parent-child relationships like inheritance, this project implements how the same method can behave differently in different classes. I used this concept to build a Company Management System where different employee roles could perform similar actions in their own way. 

This project made OOP feel much more dynamic because the same function name produced different outputs depending on the object being used.

# What I Found Interesting:

One thing I liked about this project was seeing how the same method name could behave differently depending on the object.

That small idea made the program structure feel much smarter and more flexible compared to earlier OOP projects.
It also showed me why polymorphism is considered an important concept in software design.

---

# Why I Built This Project

I built this project to understand how real applications use polymorphism to create flexible and scalable systems.

As in large software systems, different objects often need to respond differently to the same operation. This project gave me practical understanding of how that behavior is implemented using method overriding.
