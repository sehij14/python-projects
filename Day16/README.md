## Day 16 - Smart Vehicle Management System

This project was built while learning the concept of Abstraction in Object-Oriented Programming.

Until this point, most projects focused on how classes interact and inherit behavior. In this project, the focus shifted towards hiding unnecessary internal  implementation details and exposing only the important functionality to the users.

To practice this concept, I created a Smart Vehicle Management System where different vehicle types shared common behavior but implemented their own specific functionality internally. 

I created a system where all vehicles share some basic rules but each speific vehicle like a car or truck handles its own unique tasks.

Compared to previous OOP projects, I focused on hiding the messy details. Therefore the structure became cleaner and more controlled.

What This Project Helped Me Understand

One thing that became clearer during this project was the difference between:

- showing functionality to users,
- and hiding internal implementation details.

Before learning abstraction, I mostly focused on making programs work. This project introduced the idea that good software design is also about controlling how systems are accessed and structured.

---

Why I Chose This Project

Vehicle systems are useful for practicing abstraction because different vehicles can share common actions like:

- starting,
- stopping,
- displaying details,

while still implementing those actions differently internally.

That made this project a good practical example for understanding abstract classes and method implementation.
